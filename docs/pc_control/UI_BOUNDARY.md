# PCControl UI 边界冻结说明

## 1. 本文件目的

本文件用于在代码仓库内冻结 UI / 设置 / 配置边界，防止继续把全局设置、设备设置、批量设置、游戏账号设置混在一起。

本文件不代表 UI 重构已经完成，只代表 P0 阶段已经开始执行。

---

## 2. 三类设置入口

### 2.1 全局设置

全局设置只允许承载：

```text
平台附加能力
客户环境配置
本地运行环境
诊断与更新
```

允许内容包括：

```text
服务器连接
网络配置
客户外部账号数据库连接
接码平台
邮箱服务
ADB 设置
日志与诊断
更新设置
本地缓存
```

### 2.2 设备设置

设备设置只处理：

```text
单设备游戏运行配置
```

允许内容包括：

```text
主要设置
账号设置
任务设置
物品处理
购买设置
交易设置
制造设置
铸币设置
其他游戏参数
```

### 2.3 批量设置

批量设置只处理：

```text
多设备批量应用配置
```

允许内容包括：

```text
目标设备摘要
批量账号
批量任务
批量参数
差异预览
应用结果
```

---

## 3. 全局设置禁止项

全局设置中禁止出现：

```text
游戏账号列表
游戏密码表格
角色职业配置
任务路线配置
仓库物品策略
交易所策略
购买补给策略
制造策略
铸币策略
单设备参数
批量设备参数
```

如果当前历史弹窗中存在以上内容，应视为历史混合入口，后续迁移时必须移出全局设置。

---

## 4. 账号来源边界

当前已确认的游戏账号来源只有：

```text
手动录入游戏账号
客户外部账号数据库
```

严禁在当前 UI 中加入未确认账号来源。

如果未来需要平台托管游戏账号库，必须单独立项、单独设计、单独验证，不能混入当前 UI 边界重构。

---

## 5. 账号设置页边界

账号设置页里的账号是：

```text
游戏账号
```

不是：

```text
PC 登录账号
后端用户账号
授权账号
```

账号设置页属于：

```text
设备设置 / 批量设置
```

不属于：

```text
全局设置
```

---

## 6. 密码处理边界

游戏密码必须满足：

```text
默认隐藏
允许显示
允许隐藏
允许复制
允许编辑
允许受控明文导出
```

禁止：

```text
日志打印真实密码
诊断包默认包含真实密码
默认明文导出
写入 default.yaml
```

明文导出必须二次确认。

---

## 7. 设备管理页边界

设备管理页主操作工具栏应放在底部。

顶部只保留：

```text
状态
导航
筛选
```

底部主操作工具栏承载：

```text
全选
反选
清空选择
批量设置
启动脚本
停止脚本
重启脚本
激活设备
解绑设备
刷新
导出
```

禁止加入：

```text
远程控制
投屏
scrcpy
接管控制
公网远控
```

---

## 8. 旧 settings_dialog.py 冻结规则

`ui/widgets/settings_dialog.py` 是历史混合弹窗。

P0 起执行以下规则：

```text
不再新增游戏配置页
不再新增游戏账号表
不再新增游戏参数入口
不再新增物品/交易/制造/铸币功能
不再作为后续重构的主要扩展点
```

后续迁移目标：

```text
全局配置 -> ui/dialogs/global_settings_dialog.py
单设备游戏配置 -> ui/dialogs/device_settings_dialog.py
批量游戏配置 -> ui/dialogs/batch_settings_dialog.py
```
